ステップ1 Difyアカウント作成

5分

Dify公式サイトでアカウント作成

まずはDifyの公式サイトにアクセスしてアカウントを作成します。

Dify公式サイト ログイン画面

作成手順

  1. 「Sign up」をクリック
  2. メールアドレスとパスワードを入力
  3. メール認証を完了
  4. 基本情報を入力してアカウント設定完了

ステップ2 必要なAPIキーの準備

15分

OpenAI API

必須

GPT-4を使用するために必要です。記事生成のメインエンジンとして使用します。

OpenAI Platform APIキー管理

設定手順

  • OpenAI Platformにログイン
  • 「API keys」→「Create new secret key」
  • キー名を入力して生成
  • 生成されたキーをコピーして保存

Google Gemini API

必須

検索意図分析とペルソナ推定に使用します。多角的な分析を実現します。

Google AI Studio APIキー作成

設定手順

  • Google AI Studioにアクセス
  • 「Get API key」をクリック
  • 新しいプロジェクトまたは既存プロジェクトを選択
  • APIキーを生成してコピー

Anthropic Claude API

必須

最終的なリライトと品質向上に使用します。読みやすい文章に仕上げます。

Anthropic Console APIキー管理

設定手順

  • Anthropic Consoleにサインアップ
  • 「Settings」→「API Keys」
  • 「Create Key」で新しいキーを生成
  • 用途に応じた名前を付けて保存

Bright Data API

必須

検索エンジン結果取得とWebスクレイピングに使用します。競合調査に不可欠です。

Bright Data データセット

設定手順

  • Bright Dataにサインアップ
  • 「Datasets」→「Search Engine Results」を選択
  • APIアクセス設定を有効化
  • 認証情報を取得

月額コスト見積もり

OpenAI API $20-50/月
Google Gemini 無料枠あり
Claude API $15-30/月
Bright Data $500~/月
合計見積もり $535-580/月

ステップ3 Difyでのモデル・プラグイン設定

10分

AIモデル設定

取得したAPIキーをDifyに登録します。

  1. Difyダッシュボードで「Settings」→「Model Provider」
  2. OpenAI、Google、Anthropicを選択して各APIキーを入力
  3. 「Test」ボタンで接続確認
  4. すべて緑色のチェックマークが表示されることを確認

プラグイン設定

検索とスクレイピングプラグインを設定します。

  1. 「Tools」→「Browse all tools」
  2. 「Bright Data Search Engine」をインストール
  3. 「Scrape as Markdown」プラグインをインストール
  4. Bright Data認証情報を設定

重要な注意事項

  • APIキーは絶対に他人に共有しないでください
  • 定期的な利用料金の確認を行ってください
  • テスト実行時は少量のデータから始めてください

ステップ4 ワークフロー構築

30分

推奨方法：テンプレートインポート

事前に作成されたワークフローテンプレートをインポートして、すぐに使い始められます。

ワークフロー詳細を確認

手動構築

ワークフロー詳細ページを参考に、各ノードを手動で設定します。

構築手順概要

  • 「Create Workflow」で新しいワークフローを作成
  • 13個のノードを順番に配置（Start→Search→Extract→...→End）
  • 各ノードのプロンプトと設定を ワークフロー詳細 から参照
  • ノード間の変数連携を設定
  • テスト実行で動作確認

ステップ5 テスト実行・トラブルシューティング

15分

初回テスト実行

テスト用パラメータ例

keyword "リモートワーク 生産性"
persona "中小企業の人事担当者"
article_type "解説"

まずは簡単なキーワードでテスト実行を行い、全体の流れを確認しましょう。

よくある問題と解決方法

APIキーエラーが発生
  • APIキーが正しく入力されているか確認
  • 各サービスでAPIキーが有効化されているか確認
  • 利用制限や残高不足がないか確認
検索結果が取得できない
  • Bright Dataの設定を再確認
  • 検索キーワードが適切か確認
  • 地域設定やリクエスト数制限を確認
処理時間が長すぎる
  • Loopノードの反復回数を制限（最大3回）
  • 並列処理可能な部分を最適化
  • タイムアウト設定を適切に調整
