## 🎯 プロジェクト概要

### 目的
検索意図を満たし再検索を減らす「人の役に立つ高品質コンテンツ」を、AI支援で継続的・スケーラブルに量産できる仕組みを構築する。

### 適用範囲
- クロスリンクの自社メディア
- 特設LP
- 資料DL用記事群

初期は「派遣法・教育訓練・派遣会社向け業界記事」中心。将来はジャンル横展開を予定。

### 主要指標
- **自動化ステップ**: 12段階のワークフロー
- **AI モデル連携**: GPT-4、Gemini、Claude
- **月間記事目標**: 8本以上継続（短期ゴール：累計100本）

## ✨ 主要機能

### 1. 反復精査ループ
知識ギャップを自動検出し、追加リサーチで情報を補完。競合記事の80%以上を網羅する構成案を生成。

### 2. マルチAI連携
- **GPT-4**: 共起語抽出、本文生成、競合分析
- **Gemini**: クエリ分析、ペルソナ深掘り
- **Claude**: 最終リライト、トーン調整
- **GPT o3/o4**: 高精度ファクトチェック

### 3. RAG基盤
法規PDF・社内資料を知識ベース化し、根拠に基づいたファクトチェックを実現。

### 4. 品質指標
完読率・離脱率・再検索兆候を監視し、継続的な品質改善を実現。

## 🔄 ワークフロー

### 3フェーズプロセス

```
【フェーズ1：準備】
⓪ 人間：キーワード選定
↓
① Start：入力受付
↓
② Search：上位10サイト取得
↓
③ Extract：本文抽出

【フェーズ2：分析】
④ Gemini：クエリ分析・ペルソナ深掘り（Question明確化）
↓
⑤ GPT：共起語・関連キーワード抽出【心臓部】（Answer準備）
↓
⑥ GPT：競合分析・差別化戦略（フレッシュネス最優先）
↓
⑦ GPT：アウトライン生成
↓
⑧ Loop：一次情報追加・知識ギャップ解消

【フェーズ3：記事生成】
⑨ GPT：本文生成・10重チェック指示（3,000〜10,000文字）
↓
⑩ GPT o3/o4：ファクトチェック（真実/虚偽/不明瞭判定）
↓
⑪ Claude：最終リライト・品質向上
↓
⑫ End：SEO記事完成
```

### 重要原則
- **QuestionとAnswerのズレをゼロに**: 検索意図と記事内容の完全一致
- **情報のフレッシュネス最優先**: 最新データによる差別化
- **一次情報の組み込み**: 国・公的機関・銀行等の信頼できるデータ
- **10重チェック**: AIの省略を防ぐ徹底的な確認プロセス

- 各ノードの出力を検証
